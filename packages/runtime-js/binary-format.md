# Keyma binary wire format

This is the **canonical, cross-runtime** specification for Keyma's binary
serialization. It is implemented byte-identically by `@keyma/runtime-js`
(`src/binary.ts`, the reference implementation), `@keyma/runtime-python`
(`keyma/runtime/binary.py`), and `@keyma/runtime-cpp` (`include/keyma/binary.hpp`).

Byte parity across the three runtimes is the **cardinal invariant** — the shared
fixture set (`test/binary-fixtures.json`, generated by the JS reference codec and
asserted by all three suites) is the gate.

## Relationship to `serialize`

`serialize()`/`deserialize()` do **not** emit bytes — they produce a canonical wire
*record* (a JS object / Python dict / C++ `keyma::Value`). `JSON.stringify` is a
*separate* boundary. **Binary is an alternative encoder of the same per-field data,
parallel to JSON.** The codec mirrors `serialize`'s per-field *traversal* (the same
type switch, the same `dateTime`→epoch-ms / `embedded`→recurse-via-`refs` /
`array`→element-map / `reference`→bare-id conversions). The single deviation: `bytes`
is encoded **raw** (length-prefixed), skipping the JSON record's base64 step.

The codec walks the *raw model record* (the same input `serialize` receives — `Date`
objects, `Uint8Array`, nested model instances), not `serialize`'s output. It applies
the same target filtering (`client` strips `private` fields; `database` strips
`ephemeral` fields).

## Primitives

### Unsigned varint (LEB128)

A non-negative integer is encoded little-endian in 7-bit groups; the high bit (`0x80`)
of each byte is a continuation flag (set on every byte except the last). `0` encodes as
a single `0x00` byte. Used for tags/keys, lengths, counts, and zig-zagged integers.

### Zigzag (signed → unsigned)

Signed 64-bit integers are mapped to unsigned via zigzag so small-magnitude negatives
stay short: `zigzag(n) = (n << 1) ^ (n >> 63)` (arithmetic shift), then encoded as an
unsigned varint. Decode: `unzigzag(u) = (u >> 1) ^ -(u & 1)`.

### Fixed floats

`float64` is 8 bytes IEEE-754 **little-endian**; `float32` is 4 bytes IEEE-754
little-endian.

## Wire types

Each field on the wire is keyed by `varint(key)` where `key = tag * 8 + wiretype`
(`tag << 3 | wiretype`). The 3-bit wiretype tells a decoder how to read (or **skip**,
for an unknown tag) the payload:

| wiretype | name    | payload                                  |
|:--------:|---------|------------------------------------------|
| `0`      | VARINT  | one unsigned varint                      |
| `1`      | FIXED64 | 8 bytes                                  |
| `2`      | LENGTH  | `varint(len)` then `len` bytes           |
| `3`      | NULL    | *(none)* — an explicit `null` value      |
| `4`      | FIXED32 | 4 bytes                                  |

All five are self-describing, so a decoder can skip an unknown tag's payload without
knowing the field type — this is the forward/backward-compatibility (durability)
guarantee.

## Field tags

A field's **tag** is a positive integer identifying it on the wire (field *names* are
never written):

- **Phase 1 (runtime-only):** the field's **1-based declaration index** in
  `schema.fields` — i.e. `index + 1`. Target filtering (visibility/ephemeral) omits a
  field's key but **does not renumber** the survivors, so client and server agree on
  tags.
- **Phase 2 (committed manifest):** the field's `field.tag` from `keyma.tags.json`.

The codec reads `field.tag ?? (index + 1)`, so the two phases share one codec. A
freshly bootstrapped Phase-2 manifest (`1,2,3,…` in declaration order) is byte-identical
to Phase 1 for an unchanged schema.

## Record framing

A **record** is a flat sequence of fields, each `varint(key) · payload`, read until the
buffer (top-level) or the enclosing length window (embedded) is exhausted. There is **no
terminator** and **no field count** — absent fields are simply omitted, and presence is
the field's key appearing. Fields are emitted in `schema.fields` declaration order
(stable everywhere), but a decoder keys purely by tag and tolerates any order.

Encoding a field with present value `v`:

- `v === undefined` → the field is **omitted** (matches `JSON.stringify` dropping
  `undefined`).
- `v === null` → `varint(tag·8 + NULL)`, no payload.
- otherwise → `varint(tag·8 + wiretype)` then the type's payload (below).

## Per-type encoding

The wiretype and payload are a pure function of the field's `type` (and null-ness):

| IR type kind                  | wiretype | payload                                                       |
|-------------------------------|:--------:|---------------------------------------------------------------|
| `boolean`                     | VARINT   | `1` or `0`                                                     |
| `integer` (signed)            | VARINT   | `zigzag(n)`                                                    |
| `integer` (`unsigned: true`)  | VARINT   | `n` (plain LEB128)                                             |
| `bigint`                      | VARINT   | `zigzag(n)` (int64 range; mirrors the int64 runtime repr)      |
| `dateTime`                    | VARINT   | `zigzag(epochMs)` — `Date.getTime()` / epoch-ms int           |
| `number`, `bits === 32`       | FIXED32  | float32 LE                                                     |
| `number`, otherwise           | FIXED64  | float64 LE                                                     |
| `string` `id` `enum` `date` `time` `decimal` | LENGTH | UTF-8 bytes of the string value                 |
| `bytes`                       | LENGTH   | **raw** bytes (no base64)                                      |
| `embedded`                    | LENGTH   | the nested record's bytes (recurse via `refs`)                |
| `reference`                   | per `idType` | the **bare id**: VARINT(zigzag) if `idType` is a (signed) integer, else LENGTH(UTF-8). `{id}`/populated objects collapse to their `id`. |
| `array`                       | LENGTH   | `varint(count)` then, per element, `byte(wiretype) · payload` (the element token — see below) |
| `null` (any nullable field set to null) | NULL | *(none)*                                       |
| `json`                        | LENGTH   | a self-describing generic value (see below)                   |

`bits` on `integer` is **range metadata only** — it never changes the wire bytes (a
varint is width-agnostic), so `Integer<8|16|32|64>` and the matching `Unsigned<…>`
round-trip identical varint bytes. Only `number`'s `bits === 32` selects a narrower
(FIXED32) encoding.

### Array elements

An array is `LENGTH`-framed (so an unknown array-typed tag skips cleanly). Inside the
length window: `varint(count)`, then `count` **element tokens**. An element has no tag
(it is positional), so it carries a standalone 1-byte wiretype followed by the payload:
`byte(wiretype) · payload`. A `null` element (when `elementNullable`) is `byte(NULL)`
with no payload. Nested arrays and embedded elements recurse through the same element
token. The element's *type* comes from the schema (`type.of`); the wiretype byte only
distinguishes `null` from a value and drives skipping.

### References

A reference is stored as its target's **bare id** (the same value `serialize` +
reference-normalization put on the wire for writes, and what the DB stores). `idType`
(the target's `id` field type, filled in by the frontend; defaults to `string`) selects
the scalar encoding: a signed `integer` id → VARINT(zigzag); anything else → LENGTH/UTF-8.
A populated reference object collapses to its `id`. (Encoding a *fully populated*
reference sub-record over binary is deferred — a read-path enhancement; binary is for
compact transport and at-rest storage, where references are bare ids.)

### `json` (self-describing, best-effort parity)

A `json` field's value is schema-opaque, so it is encoded with a recursive,
self-describing generic value: a 1-byte kind tag then payload —
`0` null · `1` false · `2` true · `3` int (`zigzag` varint) · `4` float64 LE ·
`5` string (`varint(len)`+UTF-8) · `6` array (`varint(count)`+elements) ·
`7` object (`varint(count)`+`[varint(keyLen)+UTF-8 key, value]`) · `8` bytes
(`varint(len)`+raw). The whole thing is wrapped in the field's `LENGTH` window.
Round-trips within a runtime are exact. Cross-runtime byte parity for `json` is
**best-effort** (an integer-valued JS `number` is indistinguishable from a float, unlike
Python/C++), so `json` fields are excluded from the cross-language parity fixtures.

## Decoding

Read fields until the buffer/window is exhausted: read `varint(key)`, split into
`tag = key >> 3`, `wiretype = key & 7`. If a field with that tag exists in the schema
(`field.tag ?? index+1`), decode its payload by the field's *type* (using the wiretype
for `null`/float-width); otherwise **skip** the payload by wiretype alone (the durability
guarantee). The result mirrors `deserialize`'s output: `dateTime`→`Date`, `bytes`→
`Uint8Array`, `embedded`→constructed sub-record, `reference` bare-id→`{id}` stub, etc.

## Transport integration

`encodeBinary`/`decodeBinary` are the **per-record building blocks** a transport uses to
choose binary over JSON at the wire boundary. JSON stays the **default and fallback**:

- `createDirectTransport` is in-process and **marshal-free** — it hands the request object
  straight to the server, so no encoding is involved and it is unaffected.
- Network transports are user-supplied. A binary transport keeps the protocol envelope
  (operations / where / projections — schema-opaque metadata) as JSON and encodes the
  **schema-typed record payloads** (`create`/`update` `data`, and each result `data`) with
  `encodeBinary(schema, record, { target })`, decoding responses with `decodeBinary`. This
  is where the keyless size win lands; the envelope carries no record fields.
- Phase-1 bytes use index tags — safe for **same-version transport**. Do not persist them
  durably until Phase-2 manifest tags are frozen.

## Non-goals / deferred

- `enum` ordinals (member-order foot-gun) — enums are their canonical **string**.
- Fully-populated reference sub-records over binary (read-path; bare ids only for v1).
- Cross-runtime `json` byte parity (round-trip within a runtime is guaranteed).
